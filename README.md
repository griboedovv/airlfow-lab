# Лабораторная работа №2: Обработка потоковых данных с Apache Kafka и Airflow

## Описание проекта
Данный проект реализует систему потоковой обработки данных для сервиса Каршеринга. Система включает в себя генератор событий (состояние авто, поездки), брокер сообщений Kafka для их транспортировки, скрипт-потребитель для расчета стоимости поездок и оркестратор Airflow для управления регламентными задачами.

## Задачи работы
- Развернуть Apache Kafka в режиме KRaft на базе WSL2 (Ubuntu).
- Разработать масштабируемый скрипт поставщика данных (Kafka Producer) для имитации телеметрии автомобилей (p. 1).
- Разработать скрипт потребителя (Kafka Consumer) с механизмом валидации данных и расчета итоговой стоимости аренды (pp. 1-2).
- Интегрировать процесс в Apache Airflow для автоматизации конвейера обработки данных (p. 1).

## Архитектура решения
### 1. Генерация данных (generator.py)
Логика формирования данных вынесена в отдельный класс согласно принципам SOLID (p. 1).
Отражаемые сущности: Автомобили, пользователи, зоны парковки, тарифы (по времени/пробегу), состояние авто (уровень топлива, тех. состояние).
Формат: JSON.
### 2. Поставщик данных (producer.py)
Скрипт имитирует работу бортового компьютера автомобиля. Он собирает данные о текущей поездке, выводит их в консоль и отправляет в топик carsharing_topic (p. 1).
### 3. Потребитель данных (consumer.py)
Скрипт в реальном времени считывает сообщения из Kafka (p. 2).
Валидация: Проверка уровня топлива и состояния автомобиля.
Обработка: Расчет стоимости поездки на основе выбранного тарифа (время или дистанция).
4. Оркестрация (dags/carsharing_dag.py)
Реализован DAG в Apache Airflow, который по расписанию запускает цепочку задач: генерацию, отправку и обработку данных (p. 2).
Стек технологий
Инфраструктура: WSL2 (Ubuntu), Apache Kafka 4.2.0 (KRaft mode).
Язык разработки: Python 3.10+.
Библиотеки: kafka-python, json, random.
Оркестрация: Apache Airflow 2.10.x (p. 2).
